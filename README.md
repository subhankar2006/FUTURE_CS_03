# FUTURE_CS_03 – API Security Risk Analysis

## Overview
This repository contains my **Task 3** submission for the **Future Interns Cyber Security Internship**.

The project demonstrates API security testing using **Postman** by performing GET, POST, PUT, DELETE, and parameter-based requests on the JSONPlaceholder REST API. The responses were analyzed to identify potential security risks and recommend security best practices.

## Objectives
- Understand REST API fundamentals.
- Perform API requests using Postman.
- Analyze API responses and HTTP status codes.
- Identify potential API security risks.
- Recommend security best practices.

## Tools Used
- Postman
- JSONPlaceholder API
- REST API
- HTTP Methods (GET, POST, PUT, DELETE)

## API Endpoints Tested
- GET `/posts`
- GET `/users/1`
- GET `/posts/1`
- POST `/posts`
- PUT `/posts/1`
- DELETE `/posts/1`
